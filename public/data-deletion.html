データ削除手順（Data Deletion Instructions）

最終更新: 2026年4月24日

本ページは、スレッズスケジューラー（以下「本サービス」）利用者が自己データを削除するための手順を案内します。

本サービスの構造上の特徴
本サービスは「利用者所有モデル」を採用しており、投稿データ・認証情報・スプレッドシートのすべてが利用者自身のアカウント配下に保存されています。当方（tamago）は投稿内容・認証情報に一切アクセスできません。そのため、データ削除も利用者自身が各サービスで実施していただく形になります。

1. 利用者自身が削除するデータ

(A) GitHubリポジトリの削除

  1. GitHub にログイン
  2. 利用者所有の threads-scheduler リポジトリを開く
  3. Settings → Danger Zone → 「Delete this repository」をクリック
  4. リポジトリ名を入力して確定

→ リポジトリ内のGitHub Secrets（Threadsトークン・Google認証情報等）も同時に削除されます。

(B) Googleスプレッドシートの削除

  1. Googleドライブ を開く
  2. 「スレッズ予約」スプレッドシートを選択
  3. 右クリック → 「ゴミ箱へ移動」
  4. ゴミ箱を空にする

(C) Meta Developerアプリの削除

  1. developers.facebook.com にログイン
  2. 「マイアプリ」から該当アプリを開く
  3. 「設定」→「一般」→「アプリの削除」

→ Threadsアクセストークンも無効化されます。

(D) Google Cloud Project の削除（オプション）

  1. Google Cloud Console にログイン
  2. OAuth用に作成したプロジェクトを選択
  3. IAMと管理 → 設定 → 「プロジェクトをシャットダウン」

2. 当方（tamago）保有データの削除請求

当方が保有するデータは以下2点のみです:

  • 連絡用メールアドレス
  • アカウント名（ハンドル）

削除請求方法

以下の内容で商品ページ記載のメールアドレス宛にご連絡ください:

件名: 【データ削除請求】スレッズスケジューラー

本文:
・登録アカウント名: （ハンドル名）
・登録メールアドレス: （メール）
・削除希望: 全データ削除をお願いします

ご連絡受領後、7営業日以内に削除処理を完了し、完了通知メールをお送りします。

3. Stripe決済データの削除

決済情報はStripe社が保持しており、当方は参照権限がありません。決済情報の削除はStripe社に直接ご請求ください。
