# FAQ・トラブルシューティング

スレッズスケジューラーのよくある質問と対応手順。

---

## 運用中のトラブル

### Q1. 投稿が時刻通りに来ない

**原因**: GitHub Actions cron の仕様上、±5〜15分のズレが発生します。

**対応**:
- 通常動作です。「09:00指定 → 09:00〜09:15 に投稿」として設計されています
- 15分以上遅れる場合は Actions ログを確認
- 連続して遅延する場合は、GitHub Actions の混雑状況（https://www.githubstatus.com/）を確認

---

### Q2. スプシに書いても投稿されない

**確認ポイント**:
1. **GitHub Actions が有効か**: リポジトリ → Actions タブ → workflow が disabled になっていないか
2. **Secretsが揃っているか**: Settings → Secrets で9つのキーがすべて登録されているか
3. **スプシ権限**: サービスアカウントメアドが編集者として追加されているか
4. **日時フォーマット**: A列の投稿日時が日付型になっているか（文字列だとNG）
5. **ステータス列**: D列が「未投稿」のままか、「エラー」になっていないか

**対応**:
- リポジトリ → Actions → 「post」workflow → Run workflow で手動実行してログ確認
- ログにエラーが出ていたら該当エラーを `docs/faq.md` の対応表で探す

---

### Q3. Discord通知が来ない

**確認ポイント**:
- Webhook URL が正しいか（`DISCORD_WEBHOOK_URL` Secret）
- Discord チャンネルが削除されていないか
- Webhook URL が regenerate されていないか

**対応**:
- Discord → チャンネル設定 → 連携サービス → Webhook → 新しいURLを再発行
- GitHub Secrets を更新

---

### Q4. 「トークンが期限切れ」とエラーが出る

**原因**: Threads長期トークンは60日で失効。通常は週1の自動更新ジョブで延命される。

**対応**:
1. `refresh-token` workflow が週1で実行されているか確認
2. 失敗している場合は手動で再認可が必要（Step 5 のOAuth認可フローを再実施）
3. サポートプラン契約者はtamagoに連絡、Zoomで再認可サポート可

---

### Q5. 重複投稿された

**原因**: at-least-once配信のため稀に発生します（Container Status API の反映遅延等）。

**対応**:
- 投稿された重複ポストをThreads側で手動削除
- 発生頻度が高い場合はサポートへ連絡、運用ログから原因特定

---

## セットアップ関連

### Q6. Google OAuth consent screen で verification を要求された

**原因**: `spreadsheets` は Sensitive scope のため、Google側の判定によっては verification（本人確認審査）を求められることがある。

**対応（fallback: サービスアカウント方式）**:

1. Google Cloud Console → IAMと管理 → サービスアカウント → 「サービスアカウントを作成」
2. サービスアカウント名: `threads-scheduler-sa`
3. 権限: 付与不要（Sheets権限は対象スプシで個別に与える）
4. 作成後、キー → 「キーを追加」→「新しい鍵を作成」→ JSON
5. ダウンロードされたJSONファイル全体を GitHub Secrets `GOOGLE_SERVICE_ACCOUNT_JSON` に登録
6. サービスアカウントのメールアドレス（`...@....iam.gserviceaccount.com`）をコピー
7. 自分のスプレッドシートを開き、「共有」→ 上記メールを編集者として追加
8. コード側でOAuth方式からサービスアカウント方式に切り替え（tamagoが対応）

**注意**: サービスアカウント方式では `GOOGLE_OAUTH_REFRESH_TOKEN` `GOOGLE_OAUTH_CLIENT_ID` `GOOGLE_OAUTH_CLIENT_SECRET` は不要になります。

---

### Q7. Meta Developerアプリで Threads API が見つからない

**原因**: Use Case の選択ミス、またはMeta側の提供地域制限。

**対応**:
- アプリのUse Caseを「Business」に変更
- 「Products」→「Threads API」で検索して追加
- それでも出ない場合、地域制限の可能性あり → Metaサポートへ問い合わせ

---

### Q8. 電話番号認証のSMSが届かない

**対応**:
- 別のキャリア（docomo/au/softbank）の電話番号を試す
- IP電話・050番号は不可な場合が多い
- 海外SMSブロック設定を確認
- 時間を置いて再試行（5〜10分）

## 解約・データ削除

### Q9. サポートプランだけ解約したい

**手順**:
1. 商品ページ記載のメールアドレス宛に「サポートプラン解約希望」と送信
2. 登録アカウント名・メールアドレスを記載
3. 当月末で解約、翌月以降の課金停止
4. システム本体（リポジトリ・スプシ・Metaアプリ）は引き続き利用者所有のまま稼働継続

---

### Q10. 完全にサービス利用をやめたい

**利用者側の作業**:
1. GitHub → リポジトリ削除
2. Googleドライブ → スプシ削除
3. Meta Developer → アプリ削除
4. Google Cloud Console → OAuthクライアント削除（任意）
5. Discordチャンネル → Webhook削除または非公開化

詳細: [データ削除手順ページ](../docs/data-deletion.html)

**tamago保有データの削除**:
- 連絡用メールアドレス・アカウント名の削除を、商品ページ記載のメール宛にご請求ください
- 7営業日以内に完了通知いたします

## 技術的な質問（上級者向け）

### Q11. 複数のThreadsアカウントで運用したい

**v1では非対応**。以下の回避策あり:
- **方法A**: アカウントごとに別のGitHubリポジトリを作成（セットアップ費が人数分発生）
- **方法B**: v2での正式対応を待つ（時期未定）

---

### Q12. 投稿ネタをAIで自動生成したい

**本サービスではAI生成機能は提供していません**。別途:
- ChatGPT / Gemini / Claude 等で投稿文を生成
- 生成した文面をスプシB列に貼り付け
- 当方の別プロジェクト（AI投稿文生成）をご希望の場合は別途お問い合わせ

---

### Q13. 投稿時刻を5分単位で厳密に守りたい

**標準は10分cron**のため、最大15分のズレが発生します。

**対応**:
- 5分cron オプション（有料プラン・GitHub Actions無料枠超過のため、月額課金が別途発生）
- 利用者がGitHub Actions有料プラン加入、workflowファイルのcron設定を変更
- サポートプランで対応可

---

### Q14. 画像付き投稿を後から追加したい

**対応可能**:
1. スプシに「画像URL」列を追加
2. 「設定」シートで画像機能ON
3. Google Driveにアップした画像の共有URLを使用（ただし直リンク化が必要）
4. サポートプランで対応

---

### Q15. 投稿履歴を90日以上保存したい

**設計上は90日ローリング**ですが、利用者のスプシ内の話なので自己管理で対応可能:
- スプシの「投稿履歴」シートをコピーしてアーカイブ
- または履歴シートの自動削除ロジックをOFF（scripts/rotate-history.ts を無効化）
