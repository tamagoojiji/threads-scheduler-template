# Googleスプレッドシートテンプレート仕様

利用者に配布する「スレッズスケジューラー」テンプレートスプレッドシートの構造定義。tamagoさんがマスターテンプレートを1個作成し、申込時にコピー配布する。

---

## シート構成

| シート名 | 用途 | 利用者権限 |
|---------|------|---------|
| 投稿予約 | 利用者が投稿を登録・状態確認 | A〜D列編集可（D=画像URL） / E〜L列は保護 |
| 投稿履歴 | システムが投稿結果を記録 | 閲覧のみ |
| APIキー設定 | API認証情報の登録UI | 利用者が値入力後、PropertiesService転記 |

---

## シート「投稿予約」

### カラム定義（新レイアウト: ステータスをA列に、利用者列を左、システム列を右側非表示）

| 列 | 項目 | 型 | 利用者入力 | システム更新 | 備考 |
|---|---|---|---|---|---|
| A | ステータス | 文字列 | — | ✅ | `未投稿` / `処理中` / `投稿済` / `エラー` / `スキップ` |
| B | 日付 | 日付型 | ✅ | — | カレンダーピッカー |
| C | 時刻 | 文字列 | ✅ | — | 半角 `HH:MM` 形式で手入力（例: `09:30` / `14:00` / `23:59`）。不正値は入力時エラー |
| D | 画像URL | 文字列 / セル内画像 | ✅ | — | ルート投稿用1枚のみ。メニュー「📷 画像をアップロード」or「🔄 セル内画像をURL化」 |
| E | 投稿本文（ルート） | 文字列 | ✅ | — | 500字以内。**必須**（空欄行は処理対象外） |
| F | ツリー1 | 文字列 | ✅（任意） | — | ツリー連鎖の1段目。空欄なら単発投稿 |
| G | ツリー2 | 文字列 | ✅（任意） | — | ツリー連鎖の2段目 |
| H | ツリー3 | 文字列 | ✅（任意） | — | ツリー連鎖の3段目 |
| I | ツリー4 | 文字列 | ✅（任意） | — | ツリー連鎖の4段目（最大） |
| J | 投稿日時 | 数式 | — | — | `=IFERROR(IF(AND(NOT(ISBLANK(B2)),NOT(ISBLANK(C2))),B2+TIMEVALUE(C2),""),"")` |
| K | operation_id | 文字列 | — | ✅ | UUID（**非表示**） |
| L | attempt_count | 数値 | — | ✅ | 試行回数（**非表示**） |
| M | state_updated_at | 日時 | — | ✅ | 状態遷移時刻（**非表示**） |
| N | creation_id | 文字列 | — | ✅ | Threads container ID（**非表示**） |
| O | posted_at | 日時 | — | ✅ | 投稿確定時刻（**非表示**） |
| P | threads_post_id | 文字列 | — | ✅ | 最後に投稿したID（ツリー時は最終ツリーのID）（**非表示**） |
| Q | error_message | 文字列 | — | ✅ | 失敗時のエラー概要（**非表示**） |

### ヘッダー行（1行目）

```
ステータス | 日付 | 時刻 | 画像URL | 投稿本文 | ツリー1 | ツリー2 | ツリー3 | ツリー4 | 投稿日時 | operation_id | attempt_count | state_updated_at | creation_id | posted_at | threads_post_id | error_message
```

### ツリー機能（実装済）

- **1行 = 1ツリー（または単発投稿）**
- E列「投稿本文」が**ルート**、F〜I列「ツリー1〜4」が任意で連鎖
- 動作: ルート publish → `threads_post_id` 取得 → ツリー1 を `reply_to_id` で publish → 連鎖
- **各 publish 間に2秒sleep**（Bot判定回避）
- **任意段階で失敗 → 即停止・status=エラー・Discord通知・attempt_count=MAX固定で再試行抑止**（連続投稿防止）
- 画像はルート投稿に1枚のみ（各ツリー段には付かない仕様）

### 後方互換

- コードは**ヘッダー名で列を解決**するため、旧レイアウト（ステータスがE列等）でもツリー列が無いシートでも動作
- 旧レイアウトから新レイアウトへの移行はメニュー「🔧 シートをツリー対応レイアウトに変換」で自動化（バックアップシート自動作成）

### データ検証

- **A列（ステータス）**: プルダウン（未投稿/処理中/投稿済/エラー/スキップ）
- **B列（日付）**: 日付型、書式 `yyyy/MM/dd`
- **C列（時刻）**: 手入力。半角 `HH:MM`（00:00〜23:59）のみ許可、不正値は入力時エラー（`requireFormulaSatisfied` + REGEXMATCH）
- **J列（投稿日時）**: ARRAYFORMULA 数式・グレーアウト・触らない

### コード側の参照範囲
- **読み取り**: `投稿予約!A2:{lastCol}`（`gas/sheets.js` の `readPendingRows()`、ヘッダー名で列解決）
- **更新**: 列ごとに個別書き込み（`updateRow()`、列順序非依存）

### ツリー機能（実装済の仕様再掲）

- 上記カラム定義のとおり、**1行 = 1ツリー**。E列「投稿本文」がルート、F〜I列「ツリー1〜4」が任意の連鎖
- 動作: ルート publish → `threads_post_id` 取得 → ツリー1 を `reply_to_id` で publish → 連鎖
- 各 publish 間に2秒sleep（Bot判定回避）
- 任意段階で失敗 → 即停止・status=エラー・Discord通知・attempt_count=MAX固定で再試行抑止
- ルート publish 後の中断検出（次回実行時に creation_id PUBLISHED or history success がある場合）→ ツリー行は完了扱いせず「中断検出」エラー固定で手動対応
- 画像はルート（D列）に1枚のみ。各ツリー段には画像をつけない仕様

### ProtectedRange設定（任意）

- システム列（K〜Q）はそもそも**列非表示**になるため通常は触られない
- 念のため保護したい場合: K:Q 列を保護範囲に指定（編集権限はスクリプト実行ユーザーのみ）

### 条件付き書式（見やすさ向上）

| ステータス | 背景色 |
|---|---|
| 未投稿 | 白（デフォルト） |
| 処理中 | 薄黄色 |
| 投稿済 | 薄緑 |
| エラー | 薄赤 |
| スキップ | グレー |

## シート「投稿履歴」

append-onlyで投稿結果を記録。利用者は閲覧のみ。

### カラム定義

| 列 | 項目 | 備考 |
|---|---|---|
| A | 投稿日時（ログ時刻） | ISO8601 |
| B | operation_id | 対応する予約行のID |
| C | creation_id | Threads container ID |
| D | 本文抜粋 | 先頭50文字 |
| E | 結果 | `成功` / `エラー` |
| F | post_id | 成功時のThreads投稿ID |
| G | エラー内容 | 失敗時のエラーメッセージ |

### ヘッダー行（1行目）

```
ログ時刻 | operation_id | creation_id | 本文抜粋 | 結果 | post_id | エラー内容
```

### ProtectedRange設定

- **全シート保護** → サービスアカウントのみappend可、利用者閲覧のみ

## シート「APIキー設定」

利用者がAPI認証情報を入力するUI。`gas/setup-sheet.js` の `setupSettingsSheet()` で自動生成。

| A列（項目） | B列（値・ペースト先） | C列（状態） |
|---|---|---|
| THREADS_APP_ID | (利用者がコピペ) | 説明 / 保存後マスク |
| THREADS_APP_SECRET | 〃 | 〃 |
| THREADS_ACCESS_TOKEN | 〃 | 〃 |
| THREADS_USER_ID | 〃 | 〃 |
| DISCORD_WEBHOOK_URL | 〃 | 〃 |

メニュー「💾 設定シートからPropertiesに保存」で値を PropertiesService に転記し、B列をクリア。

> 旧仕様にあった「画像機能 ON/OFF」「タイムゾーン」「投稿制限字数」設定は廃止。
> 画像投稿の判定は「D列が公開HTTPS URL文字列か否か」のみ。セル内画像は事前にメニュー「🔄 セル内画像をURL化」での変換が必須（未変換のまま投稿時刻が来ると即エラー）。

---

## 画像投稿の保存先

- **D列の値が公開HTTPS URL（文字列）** → そのまま Threads API に渡される
- **D列にセル内画像が配置されている** → 投稿前に**メニュー「🔄 セル内画像をURL化」を実行**して URL に変換する必要あり。未変換のまま投稿時刻が来た場合、その行は即「エラー」になり Discord 通知される（リトライしない）
- **メニュー「📷 画像をアップロード」** からのファイル選択でも同フォルダに保存
- 保存先フォルダは初回利用時に自動作成 + 「リンクを知っている人は閲覧可」設定（公開設定が適用できない環境では明示エラーで停止）
- フォルダIDは **User Properties** `IMAGE_FOLDER_ID` に保持（実行Googleアカウント単位）。スプシを共同編集する複数アカウントが各自のDriveに保存できる
- tamago側ストレージは経由しない（利用者所有モデル維持）

## テンプレート作成手順（tamago側）

1. 新規Googleスプレッドシートを作成
2. タイトル: `【テンプレ】スレッズスケジューラー`
3. 3シート（投稿予約・投稿履歴・設定）を上記仕様で作成
4. データ検証・条件付き書式・ProtectedRange を設定
5. 「ファイル」→「共有」→「リンクを知っている全員が閲覧可」
6. URL を `https://docs.google.com/spreadsheets/d/<SHEET_ID>/edit` 形式で保存
7. 利用者申込時はこのURLを送付、「ファイル > コピーを作成」でご自身のドライブへ複製してもらう

## 利用者への配布後の設定

- 利用者のスプシに tamagoのサービスアカウントメールアドレスを「編集者」として追加
- （サービスアカウント方式に切り替えた場合）
