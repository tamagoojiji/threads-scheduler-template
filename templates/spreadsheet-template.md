# Googleスプレッドシートテンプレート仕様

利用者に配布する「スレッズスケジューラー」テンプレートスプレッドシートの構造定義。tamagoさんがマスターテンプレートを1個作成し、申込時にコピー配布する。

---

## シート構成

| シート名 | 用途 | 利用者権限 |
|---------|------|---------|
| 投稿予約 | 利用者が投稿を登録・状態確認 | A〜D列編集可（D=画像URL） / E〜L列は保護 |
| 投稿履歴 | システムが投稿結果を記録 | 閲覧のみ |
| APIキー設定 | API認証情報の登録UI | 利用者が値入力後、PropertiesService転記 |

---

## シート「投稿予約」

### カラム定義（B方式: A=日付、B=時刻、M=投稿日時の数式）

| 列 | 項目 | 型 | 利用者入力 | システム更新 | 備考 |
|---|---|---|---|---|---|
| A | 日付 | 日付型 | ✅ | — | カレンダーピッカー |
| B | 時刻 | 文字列 | ✅ | — | 30分刻みプルダウン（00:00〜23:30） |
| C | 投稿本文 | 文字列 | ✅ | — | 500字以内（超過は実行時スキップ） |
| D | 画像URL | 文字列 / セル内画像 | ✅ | — | メニュー「📷 画像をアップロード」でDriveに保存→URL自動入力 / セル内画像ドラッグ後「🔄 セル内画像をURL化」で一括変換も可 |
| E | ステータス | 文字列 | — | ✅ | `未投稿` / `処理中` / `投稿済` / `エラー` / `スキップ` |
| F | operation_id | 文字列 | — | ✅ | UUID（一度生成したら不変） |
| G | attempt_count | 数値 | — | ✅ | 試行回数（最大3） |
| H | state_updated_at | 日時（ISO8601） | — | ✅ | 状態遷移時刻 |
| I | creation_id | 文字列 | — | ✅ | Threads container ID |
| J | posted_at | 日時（ISO8601） | — | ✅ | 投稿確定時刻 |
| K | threads_post_id | 文字列 | — | ✅ | Threads投稿ID |
| L | error_message | 文字列 | — | ✅ | 失敗時のエラー概要 |
| M | 投稿日時 | 数式 | — | — | `=IFERROR(IF(AND(NOT(ISBLANK(A2)),NOT(ISBLANK(B2))),A2+TIMEVALUE(B2),""),"")` |
| N | ツリーID | 文字列 | ✅ | — | **将来用・予約**。同じツリーに属する投稿のグループID（空欄=単発投稿） |
| O | ツリー内順序 | 数値 | ✅ | — | **将来用・予約**。0=ルート / 1, 2, ... = 返信の順 |

### ヘッダー行（1行目）

```
日付 | 時刻 | 投稿本文 | 画像URL | ステータス | operation_id | attempt_count | state_updated_at | creation_id | posted_at | threads_post_id | error_message | 投稿日時 | ツリーID | ツリー内順序
```

> ⚠️ コード（`gas/sheets.js`）はヘッダー名で列を解決するため、ツリーID/ツリー内順序が無くても動作する。
> 列の追加・並び替えはヘッダー名さえ維持されれば自由（必須ヘッダー: 日付/時刻/投稿本文/画像URL/ステータス/operation_id/attempt_count/state_updated_at/creation_id/posted_at/threads_post_id/error_message/投稿日時）。

### データ検証

- **A列（日付）**: `requireDate()`、書式 `yyyy/MM/dd`
- **B列（時刻）**: 30分刻みのプルダウン（`requireValueInList`）、書式 `@`（文字列）
- **E列（ステータス）**: プルダウン（未投稿/処理中/投稿済/エラー/スキップ）
- **M列（投稿日時）**: 数式・グレーアウト・触らない

### コード側の参照範囲
- **読み取り**: `投稿予約!A2:{lastCol}`（`gas/sheets.js` の `readPendingRows()`、ヘッダー名で列解決）
- **更新**: 列ごとに個別書き込み（`updateRow()`、列順序非依存）

### ツリー機能（将来実装）

- N列「ツリーID」/ O列「ツリー内順序」を入力すると、**同ツリーの順序0をルート投稿として publish → その `threads_post_id` を順序1の `reply_to_id` に設定 → 連鎖publish** する想定
- 実装は `runner.js` の `processRow()` に分岐を追加するのみ（現在の単発投稿ロジックはそのまま流用）
- 画像URL列は各投稿（行）ごとに独立して設定可能

### ProtectedRange設定

- **E:L列（システム更新列）** を保護範囲として設定（**D=画像URL は利用者編集可**）
- 編集権限: スクリプト実行ユーザーのみ
- 利用者が誤編集しようとすると警告表示（完全ブロックではなく警告）

### 条件付き書式（見やすさ向上）

| ステータス | 背景色 |
|---|---|
| 未投稿 | 白（デフォルト） |
| 処理中 | 薄黄色 |
| 投稿済 | 薄緑 |
| エラー | 薄赤 |
| スキップ | グレー |

## シート「投稿履歴」

append-onlyで投稿結果を記録。利用者は閲覧のみ。

### カラム定義

| 列 | 項目 | 備考 |
|---|---|---|
| A | 投稿日時（ログ時刻） | ISO8601 |
| B | operation_id | 対応する予約行のID |
| C | creation_id | Threads container ID |
| D | 本文抜粋 | 先頭50文字 |
| E | 結果 | `成功` / `エラー` |
| F | post_id | 成功時のThreads投稿ID |
| G | エラー内容 | 失敗時のエラーメッセージ |

### ヘッダー行（1行目）

```
ログ時刻 | operation_id | creation_id | 本文抜粋 | 結果 | post_id | エラー内容
```

### ProtectedRange設定

- **全シート保護** → サービスアカウントのみappend可、利用者閲覧のみ

## シート「APIキー設定」

利用者がAPI認証情報を入力するUI。`gas/setup-sheet.js` の `setupSettingsSheet()` で自動生成。

| A列（項目） | B列（値・ペースト先） | C列（状態） |
|---|---|---|
| THREADS_APP_ID | (利用者がコピペ) | 説明 / 保存後マスク |
| THREADS_APP_SECRET | 〃 | 〃 |
| THREADS_ACCESS_TOKEN | 〃 | 〃 |
| THREADS_USER_ID | 〃 | 〃 |
| DISCORD_WEBHOOK_URL | 〃 | 〃 |

メニュー「💾 設定シートからPropertiesに保存」で値を PropertiesService に転記し、B列をクリア。

> 旧仕様にあった「画像機能 ON/OFF」「タイムゾーン」「投稿制限字数」設定は廃止。
> 画像投稿の判定は「D列が公開HTTPS URL文字列か否か」のみ。セル内画像は事前にメニュー「🔄 セル内画像をURL化」での変換が必須（未変換のまま投稿時刻が来ると即エラー）。

---

## 画像投稿の保存先

- **D列の値が公開HTTPS URL（文字列）** → そのまま Threads API に渡される
- **D列にセル内画像が配置されている** → 投稿前に**メニュー「🔄 セル内画像をURL化」を実行**して URL に変換する必要あり。未変換のまま投稿時刻が来た場合、その行は即「エラー」になり Discord 通知される（リトライしない）
- **メニュー「📷 画像をアップロード」** からのファイル選択でも同フォルダに保存
- 保存先フォルダは初回利用時に自動作成 + 「リンクを知っている人は閲覧可」設定（公開設定が適用できない環境では明示エラーで停止）
- フォルダIDは **User Properties** `IMAGE_FOLDER_ID` に保持（実行Googleアカウント単位）。スプシを共同編集する複数アカウントが各自のDriveに保存できる
- tamago側ストレージは経由しない（利用者所有モデル維持）

## テンプレート作成手順（tamago側）

1. 新規Googleスプレッドシートを作成
2. タイトル: `【テンプレ】スレッズスケジューラー`
3. 3シート（投稿予約・投稿履歴・設定）を上記仕様で作成
4. データ検証・条件付き書式・ProtectedRange を設定
5. 「ファイル」→「共有」→「リンクを知っている全員が閲覧可」
6. URL を `https://docs.google.com/spreadsheets/d/<SHEET_ID>/edit` 形式で保存
7. 利用者申込時はこのURLを送付、「ファイル > コピーを作成」でご自身のドライブへ複製してもらう

## 利用者への配布後の設定

- 利用者のスプシに tamagoのサービスアカウントメールアドレスを「編集者」として追加
- （サービスアカウント方式に切り替えた場合）
