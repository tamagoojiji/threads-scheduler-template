セットアップ手順書

スレッズスケジューラーを動かすまでの全ステップ

所要時間: 約90分。Zoom同伴で一緒に進めます。詰まったらすぐお声がけください。

進行ステップ（取得した値はその都度GitHub Secretsに登録します）

  1. 事前にご準備いただくもの
  2. GitHubアカウント（未登録の方）
  3. GitHubリポジトリ複製（Secrets登録の置き場を先に用意）
  4. Meta Developerアプリ作成 → THREADS_APP_ID / THREADS_APP_SECRET を登録
  5. Threads長期アクセストークン取得 → THREADS_ACCESS_TOKEN / THREADS_USER_ID を登録
  6. Google Cloud OAuth設定 → GOOGLE_OAUTH_CLIENT_ID / GOOGLE_OAUTH_CLIENT_SECRET を登録
  7. Refresh Token取得 → GOOGLE_OAUTH_REFRESH_TOKEN を登録
  8. Googleスプシのコピー → SHEET_ID を登録
  9. Discord Webhook作成 → DISCORD_WEBHOOK_URL を登録
  10. テスト投稿で動作確認

1. 事前にご準備いただくもの

  • Googleアカウント（個人でも業務でも可）
  • Threadsアカウント（投稿先）
  • PCのブラウザでThreadsにログイン済みの状態にする — threads.net を開いてログインしておく（Step 5-3 の招待承認作業をPCで行うため）
  • Discordアカウント（エラー通知受信用）
  • 電話番号（Meta Developer登録のSMS認証で必須・Facebookアカウントと連携している電話番号をご用意ください・IP電話/050は不可な場合あり）
  • Zoomで画面共有できるPC環境

📝 事前準備フォーム（推奨）

下のボタンから事前準備フォームを開いて入力してください。送信するとあなたのGoogleドライブにスプレッドシートが自動作成されます。

📝 事前準備フォームを開く

  • 各サービスのログイン確認URL付き → ログインできれば該当のID入力をパス可能
  • 入力中はブラウザに自動保存（リロードしても消えない）
  • 送信時にGoogleログイン → あなたのドライブにスプシ自動作成
  • 提供者には1文字も届きません
  • スマホでも入力可（ホーム画面に追加してアプリ化も可）
パスワード・カード情報は一切お伺いしません。すべてご自身の画面で操作いただきます。
パスワードはお手元の手書きメモ、または各ブラウザ標準のパスワード保存機能（Keychain・Chrome）でご管理ください。フォームにもパスワード欄は意図的に作っていません。

2. GitHubアカウント（未登録の方のみ）

手順
  1. github.com を開く
  2. 右上の Sign up をクリック
  3. メール・パスワード・ユーザー名を順に入力（ユーザー名は半角英数）
  4. 登録メール宛のリンクで認証を完了

3. GitHubリポジトリ複製（Secrets置き場を先に作る）

取得した値はその都度 GitHub Secrets に登録していくので、先にリポジトリを作っておきます。

手順
  1. GitHub にログイン
  2. tamagoojiji/threads-scheduler-template を開く
  3. 右上の 緑色の「Use this template」ボタン をクリック
  4. ドロップダウンから 「Create a new repository」 を選択
  5. 下表のとおり設定を入力
  6. 「Create repository」をクリック

設定値

項目設定値備考
Ownerご自身のGitHubユーザー名デフォルトのままでOK
Repository name任意（例: my-threads-scheduler）半角英数とハイフン
Description空欄でOKあとで変更可能
公開設定Private（推奨）運用カスタマイズを非公開に保てる
Include all branchesチェック不要mainブランチのみで十分

Secrets登録ページをブックマーク

手順
  1. 作成したご自身のリポジトリを開く
  2. 上部メニューの 「Settings」（歯車アイコン）をクリック
  3. 左メニューから Secrets and variables → Actions を選択
  4. このページをブックマーク → 以降の各ステップで取得した値をここに登録します
リポジトリURL（https://github.com/<ユーザー名>/<リポジトリ名>）と Secrets 登録ページをブックマークしておくと便利です。
「Use this template」ボタンが見つからない場合、ブラウザを再読み込み（Cmd+Shift+R / Ctrl+F5）してみてください。

4. Meta Developerアカウント・アプリ作成

電話番号認証では Facebookアカウントと連携している電話番号 をご利用ください。連携していない番号だと認証が通らない or アカウントロックの原因になることがあります。
手順
  1. developers.facebook.com へログイン（既存のFacebook/Instagramアカウントでログイン）
  2. 電話番号認証 → 開発者規約に同意
  3. マイアプリ → アプリを作成
  4. ユースケースで 「Threads APIにアクセス」 を選択
  5. アプリ名は任意（例: my-threads-scheduler）

アプリの設定 > ベーシック

新UIでは保存時に「フォームを保存できません」エラーが頻発します。以下すべてを揃えて初めて保存できます。
  • カテゴリ: ビジネス・ページ を選択（他カテゴリだと保存不可）
  • プライバシーポリシーURL: https://www.facebook.com/privacy/policy/ でも可
  • 利用規約URL: 任意のURL（例: https://www.google.com/）
  • アプリドメイン・アイコン: 空欄でも可

ユースケース > カスタマイズ > 設定

  • コールバックURLをリダイレクト: https://www.google.com/
  • アンインストールコールバックURL: https://www.google.com/
  • 削除コールバックURL: https://www.google.com/
3箇所すべてに入力してください。1つでも空欄だと保存エラーになります。

権限

  • threads_basic
  • threads_content_publish
他の権限（threads_manage_replies等）は不要です。

4-X. GitHub Secretsへ登録（このステップで取得した値）

Meta Developerのアプリ設定 → 基本設定 から以下を取得し、Step 3でブックマークしたSecretsページに登録します。

NameSecret（値）
THREADS_APP_IDアプリID（数字の羅列）
THREADS_APP_SECRETアプリシークレット（「表示」をクリックして取得）
登録手順: Secretsページ → 「New repository secret」 → Name と Secret を入力 → 「Add secret」

5. Threads長期アクセストークン取得

Meta Developerの公式ボタン1クリックで60日有効のトークンが取れます。OAuth認可URLやcurlは不要です。

5-1. ユーザートークン生成ツールを開く

手順
  1. Meta Developerダッシュボードで対象アプリを開く
  2. 左メニューから ユースケース セクションを展開
  3. Threads APIにアクセス → カスタマイズ をクリック
  4. 表示されたサブメニューから 「設定」 をクリック
  5. 右側ペインを下方向にスクロール → 「ユーザートークン生成ツール（User token generator）」 セクションを見つける

5-2. Threadsテスターとしてアカウントを追加

手順
  1. 「ユーザートークン生成ツール」セクション内の 「Threadsテスターを追加（Add Threads Tester）」 ボタンをクリック
  2. ポップアップで対象のThreadsアカウント名を検索（例: @tamago_app）
  3. 該当アカウントが表示されたら「追加」or「Submit」をクリック
  4. 状態は「保留中（Pending）」となり、Threads側からの承認待ち状態になる

5-3. Threads側で招待を承認（即時で完了）

Step 5-2で送信した招待を、Threads側で承認します。基本1クリックで完了します。

手順
  1. Meta Developer で「Threadsテスターを追加」→ 招待送信（Step 5-2で完了済み）
  2. 同じアカウントでログイン中の Threadsアプリ または Threads Web を開く
  3. 右下プロフィール → 設定 → アカウント → ウェブサイトの権限
  4. 「招待」セクションで「承認」をタップ
  5. 即時で「保留中 → 承認済み」に変わる
基本1クリックで完了。承認反映まで数秒〜数分。
招待が表示されない場合: Threadsアプリで上から下にスワイプして再読み込み、または上記の設定画面を直接開いてください。
招待の表示まで 5〜10分かかることがあります。

5-4. 長期アクセストークンを生成

手順
  1. 承認後、Meta Developer の「ユーザートークン生成ツール」セクションに戻る
  2. 必要であればページを再読み込み
  3. 承認済みアカウントの右側に 「アクセストークンを生成（Generate access token）」 ボタンが表示される
  4. クリック → ポップアップで「許可（Allow / Continue）」
  5. 長期アクセストークン（60日有効）が表示される
  6. 「コピー」ボタンをクリックしてコピー → 安全な場所に保管
取得する値: THREADS_ACCESS_TOKEN
形式: THAA で始まる長い英数字（200〜300文字程度）

5-5. Threads User ID取得（ブラウザのみ）

ターミナル不要。ブラウザのアドレスバーに直接入力するだけで取得できます。

手順
  1. ブラウザを開いて、アドレスバーに以下を貼り付け（<長期トークン> の部分を Step 5-4 で取得した実際のトークンに置き換え）
https://graph.threads.net/v1.0/me?fields=id&access_token=<長期トークン>
  1. Enter キーを押す
  2. ブラウザに以下のようなJSONが表示される:
{"id":"35107700672178457"}
  1. id の値（数字）が THREADS_USER_ID
取得する値: THREADS_USER_ID（数字の羅列、17桁前後）
JSONが見づらい場合: ブラウザに「JSON Viewer」拡張を入れると整形表示されます。なくても "id":"..." の数字部分をコピーすればOK。

5-X. GitHub Secretsへ登録（このステップで取得した値）

NameSecret（値）
THREADS_ACCESS_TOKENStep 5-4 で取得した長期トークン（THAAで始まる）
THREADS_USER_IDStep 5-5 で取得した User ID（17桁前後の数字）

5-6. PATの作成・登録（トークン自動更新のため）

長期トークンは60日で失効するため、毎週日曜に自動更新します。自動更新には GitHub Personal Access Token (PAT) が必要です。

手順
  1. GitHub Fine-grained PAT 作成ページ を開く
  2. 下表のとおり設定を入力
  3. 「Generate token」をクリック
  4. 表示されたトークン（github_pat_ で始まる）をコピー

PAT設定値

項目設定値
Token nameThreads Scheduler Token Refresh
Expiration1年（最大）— 期限切れの前に再作成が必要
Repository accessOnly select repositories → Step 3で作成したご自身のリポジトリを選択
Repository permissions → SecretsRead and write
その他のpermissionsすべて No access のまま

PATをGitHub Secretsへ登録

NameSecret（値）
PAT_FOR_SECRETS取得したPAT（github_pat_ で始まる文字列）
これで毎週日曜9:00（JST）に自動でトークン更新 → Discord通知 → Secretsの自動上書きが動くようになります。
PATは漏洩リスクがあるため、Repository permissions は Secrets read/write のみ に絞り、それ以外は no access のままにしてください。1年経つ前に再作成・更新してください。
登録手順: Secretsページ → 「New repository secret」 → Name と Secret を入力 → 「Add secret」

6. Google Cloud OAuth設定

手順
  1. Google Cloud Console へログイン
  2. 新規プロジェクト作成（名前: Threads Scheduler 等）
  3. APIとサービス → ライブラリ → Google Sheets API を有効化
  4. OAuth同意画面を設定:
    • User Type: External
    • App name: Threads Scheduler
    • サポートメール・デベロッパーメール: ご自身のGmail
    • スコープに .../auth/spreadsheets を追加
    • Test users にご自身のGmailを追加
  5. 「アプリを公開（PUBLISH APP）」→「本番環境にプッシュ」をクリック
必ず「本番環境にプッシュ」してください。Testing状態のまま運用するとRefresh Tokenが7日で失効します。

アプリを本番環境に公開する手順（詳細）

手順
  1. OAuth同意画面ページ を開く
  2. 該当プロジェクトを選択
  3. 「APIとサービス → OAuth同意画面」
  4. 「アプリの公開ステータス」セクション → 「アプリを公開」ボタンをクリック
  5. 確認ダイアログで「本番環境に公開」または「確認」をクリック
これで誰でも認証できるようになります（テストユーザー登録不要）。

「アクセスをブロック」エラーが出た場合

「エラー 403: access_denied / アプリは Google の審査プロセスを完了していません」 と出る場合、上記「本番環境に公開」がまだ完了していません。再度「アプリを公開」を実行してください。

「未確認のアプリ」警告が出た場合

本番環境公開後、最初のログイン時に「このアプリは Google で確認されていません」と表示されます:
  1. 警告画面で「詳細」をクリック
  2. 「(プロジェクト名)（安全ではないページ）に移動」をクリック
  3. 通常通り「許可」して進む
個人利用 / 少人数利用では verification 不要です。

OAuth Client作成

手順
  1. 左メニューから 認証情報 を開く
  2. 上部の 「+ 認証情報を作成」→「OAuthクライアントID」 をクリック
  3. アプリケーションの種類: ウェブアプリケーション
  4. 名前: Threads Scheduler 等（任意）
  5. 承認済みのリダイレクトURI: https://developers.google.com/oauthplayground を追加
  6. 「作成」をクリック

Client ID と Client Secret の取得

作成直後に表示される場合（推奨）
  1. 「作成」をクリックすると完了ダイアログがポップアップ表示される
  2. 「クライアント ID」「クライアント シークレット」の2つが表示される
  3. それぞれの右側にある コピーアイコン をクリックしてコピー → 安全な場所に保存
  4. 「JSONをダウンロード」でファイルとして取得することも可能
後から再表示する場合
  1. 認証情報ページ を開く
  2. 「OAuth 2.0 クライアント ID」の一覧から、作成したクライアント名（例: Threads Scheduler）をクリック
  3. 詳細画面の右側に 「クライアント シークレット」 セクションが表示される
  4. 値の右側のコピーアイコンをクリック
取得する値:
  • GOOGLE_OAUTH_CLIENT_ID — ...apps.googleusercontent.com で終わる文字列
  • GOOGLE_OAUTH_CLIENT_SECRET — GOCSPX- で始まる英数字

7. Refresh Token取得

手順
  1. Google OAuth Playground を開く
  2. 右上の歯車アイコン → Use your own OAuth credentials にチェック
  3. Client ID / Client Secret を入力
  4. 左側の入力欄に https://www.googleapis.com/auth/spreadsheets を貼り付け
  5. Authorize APIs をクリック → Googleログイン → 「許可」
  6. 「同意画面」で 「強制的に承認させる（Force prompt）」 オプションを有効化（Refresh Tokenを必ず返してもらうため）
  7. Exchange authorization code for tokens をクリック
  8. 表示された Refresh token（1// で始まる文字列）をコピー保存
取得する値: GOOGLE_OAUTH_REFRESH_TOKEN

7-X. GitHub Secretsへ登録（このステップで取得した値）

NameSecret（値）
GOOGLE_OAUTH_REFRESH_TOKEN1// で始まる長い文字列
登録手順: Secretsページ → 「New repository secret」 → Name と Secret を入力 → 「Add secret」

9. Discord Webhook作成

エラー・成功通知を受け取るチャンネルを作ります。

9-1. 開発者モードを有効化（先にこれを実施）

手順
  1. Discordアプリ（またはブラウザ版）を開く
  2. 画面左下の ユーザー名の右側にある歯車アイコン（ユーザー設定）をクリック
  3. 左メニューを下にスクロール → APP設定 セクションの 「詳細設定」 をクリック
  4. 「開発者モード」 のトグルをONに
  5. 左上の「✕」または「Esc」で設定画面を閉じる
これで右クリックメニューから「IDをコピー」「ウェブフックを作成」などが使いやすくなります。

9-2. 通知用チャンネル作成

手順
  1. 既存サーバーで右クリック → 「チャンネルを作成」（新規サーバーから作る場合は左サイドバー一番下の「+」）
  2. チャンネル種別: テキストチャンネル
  3. 名前: threads-scheduler-test など分かりやすい名前
  4. 「チャンネルを作成」をクリック

9-3. Webhook URLを取得

手順
  1. 作成したチャンネル名の右の 歯車アイコン（チャンネルを編集）をクリック
  2. 左メニューから 連携サービス を選択
  3. 「ウェブフック」→「ウェブフックを作成」
  4. 名前を Threads Scheduler 等に設定
  5. 「ウェブフックURLをコピー」をクリック
取得する値: DISCORD_WEBHOOK_URL
形式: https://discord.com/api/webhooks/...

10. テスト投稿で動作確認

手順
  1. スプシ「投稿予約」シートのA列に「3分後の日時」、B列に「テスト投稿」を入力
  2. D列ステータスは 未投稿 を選択
  3. GitHubリポジトリ → Actions → post ワークフロー → Run workflow
  4. 実行ログで「処理対象: 1行 → 完了」を確認
  5. Threadsアカウントで投稿が反映されたか確認
  6. スプシのD列ステータスが 投稿済 に変わったか確認
  7. Discordに ✅ 投稿成功 通知が届いたか確認
全部✅になればセットアップ完了です。お疲れさまでした。
