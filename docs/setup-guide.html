セットアップ手順書

スレッズスケジューラーを動かすまでの全ステップ（Google Apps Script版・GitHub不要）

所要時間: 約60〜90分。Zoom同伴で一緒に進めます。詰まったらすぐお声がけください。

進行ステップ

  1. 事前にご準備いただくもの
  2. Meta Developerアカウント・アプリ作成
  3. Threads長期アクセストークン取得
  4. Threads User ID取得
  5. Discord Webhook作成
  6. Googleスプシのコピー
  7. 初回セットアップ（Secrets登録）
  8. トリガーをインストール
  9. テスト投稿で動作確認

1. 事前にご準備いただくもの

サービス名をタップするとログイン画面が開きます。ログイン状態を確認しておくと当日スムーズです。

  • Googleアカウント（個人でも業務でも可）
  • Threadsアカウント（投稿先）
  • Discordアカウント（エラー通知受信用・ブラウザで開く）
  • 電話番号（Meta Developer登録のSMS認証で必須・Facebookアカウントと連携している電話番号をご用意ください・IP電話/050は不可な場合あり）
  • Zoomで画面共有できるPC環境
パスワード・カード情報は一切お伺いしません。すべてご自身の画面で操作いただきます。
GitHubアカウントは不要です。すべてGoogleアカウントのみで完結します。

2. Meta Developerアカウント・アプリ作成

電話番号認証では Facebookアカウントと連携している電話番号 をご利用ください。連携していない番号だと認証が通らない or アカウントロックの原因になることがあります。
手順
  1. developers.facebook.com へログイン（既存のFacebook/Instagramアカウントでログイン）
  2. 電話番号認証 → 開発者規約に同意
  3. マイアプリ → アプリを作成
  4. ユースケースで 「Threads APIにアクセス」 を選択
  5. アプリ名は任意（例: my-threads-scheduler）

アプリの設定 > ベーシック

新UIでは保存時に「フォームを保存できません」エラーが頻発します。以下すべてを揃えて初めて保存できます。
  • カテゴリ: ビジネス・ページ を選択（他カテゴリだと保存不可）
  • プライバシーポリシーURL: https://www.facebook.com/privacy/policy/ でも可
  • 利用規約URL: 任意のURL（例: https://www.google.com/）
  • アプリドメイン・アイコン: 空欄でも可

ユースケース > カスタマイズ > 設定

  • コールバックURLをリダイレクト: https://www.google.com/
  • アンインストールコールバックURL: https://www.google.com/
  • 削除コールバックURL: https://www.google.com/
3箇所すべてに入力してください。1つでも空欄だと保存エラーになります。

権限

  • threads_basic
  • threads_content_publish
他の権限（threads_manage_replies等）は不要です。

取得する値

名前取得元
THREADS_APP_IDMeta Developer アプリの基本設定（数字の羅列）
THREADS_APP_SECRETMeta Developer アプリの基本設定（「表示」をクリック）
これらは Step 7 でスプシのメニューから入力します。手元のメモ帳などに控えておいてください。

3. Threads長期アクセストークン取得

Meta Developerの公式ボタン1クリックで60日有効のトークンが取れます。

3-1. ユーザートークン生成ツールを開く

手順
  1. Meta Developerダッシュボードで対象アプリを開く
  2. 左メニューから ユースケース → Threads APIにアクセス → カスタマイズ → 設定
  3. 右側ペインを下方向にスクロール → 「ユーザートークン生成ツール」 セクション

3-2. Threadsテスターとしてアカウントを追加

手順
  1. 「Threadsテスターを追加」ボタンをクリック
  2. 対象のThreadsアカウント名を検索（例: @tamago_app）
  3. 「追加」をクリック → 状態は保留中（Pending）になる

3-3. Threads側で招待を承認（即時で完了）

手順
  1. 同じアカウントでログイン中の Threadsアプリ または Threads Web を開く
  2. 右下プロフィール → 設定 → アカウント → ウェブサイトの権限
  3. 「招待」セクションで「承認」をタップ
  4. 即時で「保留中 → 承認済み」に変わる
招待が表示されない場合: 5〜10分待ってから再確認、または Threadsアプリで上から下にスワイプして再読み込み。

3-4. 長期アクセストークンを生成

手順
  1. Meta Developer の「ユーザートークン生成ツール」に戻る → ページ再読み込み
  2. 承認済みアカウントの右側「アクセストークンを生成」をクリック
  3. ポップアップで「許可」
  4. 表示された長期アクセストークン（60日有効）を コピーボタンで取得 → 安全な場所に保管
取得する値: THREADS_ACCESS_TOKEN
形式: THAA で始まる長い英数字（200〜300文字程度）

4. Threads User ID取得（ブラウザのみ）

ターミナル不要。ブラウザのアドレスバーに貼り付けるだけ。

手順
  1. ブラウザのアドレスバーに以下を貼り付け（<長期トークン> を Step 3-4 で取得した値に置き換え）
https://graph.threads.net/v1.0/me?fields=id&access_token=<長期トークン>
  1. Enter キー → JSONが表示される:
{"id":"35107700672178457"}
  1. id の値（数字）が THREADS_USER_ID
取得する値: THREADS_USER_ID（17桁前後の数字）

5. Discord Webhook作成

エラー・成功通知を受け取るチャンネルを作ります。

5-1. 開発者モードを有効化

手順
  1. Discordアプリ（またはブラウザ版）を開く
  2. 左下のユーザー設定（歯車アイコン）
  3. 左メニュー APP設定 → 詳細設定 → 開発者モードON

5-2. 通知用チャンネル作成

手順
  1. 既存サーバー or 新規サーバーで「チャンネルを作成」
  2. テキストチャンネル / 名前: threads-scheduler 等

5-3. Webhook URLを取得

手順
  1. 作成したチャンネル名右の歯車 → 連携サービス → ウェブフック → ウェブフックを作成
  2. 名前を「Threads Scheduler」等に
  3. 「ウェブフックURLをコピー」
取得する値: DISCORD_WEBHOOK_URL
形式: https://discord.com/api/webhooks/...

6. Googleスプシのコピー

tamagoが用意したマスタースプシを、ご自身のGoogleドライブにコピーします。Apps Scriptコードもまとめてコピーされます。

手順
  1. tamagoが共有するマスタースプシURLを開く（Zoom中にチャットで送付）
  2. ファイル → コピーを作成 でご自身のGoogleドライブに複製
  3. コピー後、画面上部メニューに 「スレッズスケジューラー」 が表示されることを確認
「スレッズスケジューラー」メニューが表示されない場合: ページを再読み込み（Cmd+R / F5）してください。

7. 初回セットアップ（Secrets登録）

取得した5つの値をスプシに登録します。

手順
  1. スプシメニュー 「スレッズスケジューラー → 🔧 初回セットアップ（Secrets登録）」
  2. 初回は権限承認ダイアログが出るので「許可」
  3. 5つのプロンプトが順番に表示されるので、それぞれ値を入力

登録する5つの値

Name値の出どころ
THREADS_APP_IDStep 2 で取得
THREADS_APP_SECRETStep 2 で取得
THREADS_ACCESS_TOKENStep 3-4 で取得
THREADS_USER_IDStep 4 で取得
DISCORD_WEBHOOK_URLStep 5-3 で取得
登録後、「🔍 設定状況を確認」で全項目が ✅ になっていればOK。

8. トリガーをインストール

「10分ごとの投稿チェック」と「毎週日曜のトークン更新」を自動化します。

手順
  1. スプシメニュー 「スレッズスケジューラー → ⏰ トリガーをインストール」
  2. 初回は権限承認ダイアログが出るので「許可」
  3. 「トリガーをインストールしました」のダイアログが出れば完了
これで完全自動運用開始。利用者の手動操作は今後不要です。

9. テスト投稿で動作確認

手順
  1. 「投稿予約」シートに新しい行を追加:
    • A列: 今日の日付（カレンダー）
    • B列: 1分前の時刻（プルダウン or 直接入力）
    • C列: テスト投稿
    • E列: 未投稿（プルダウン）
  2. M列に投稿日時が自動表示されることを確認
  3. スプシメニュー 「スレッズスケジューラー → ▶️ 今すぐ投稿チェック」 をクリック
  4. 数秒で完了 → Threadsに投稿が反映されたか確認
  5. スプシのE列ステータスが 投稿済 になったか確認
  6. Discordに ✅ 投稿成功 通知が届いたか確認
全部✅になればセットアップ完了です。お疲れさまでした。
以降、A列・B列・C列に予定を入力するだけで、10分以内に自動投稿されます。
