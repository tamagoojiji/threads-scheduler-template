03
🔧 入力フォーム理解

初回セットアップで5項目を入力

メニュー 🔧 初回セットアップ を実行すると、5つの値を順番に聞かれます。 この手順書では、1項目ずつ取得 → その場で入力 していきます。空欄でOKを押せば既存値は保持されます。

キー名 取得場所 形式
THREADS_APP_ID Step 4 — Meta Developer アプリ作成 数字(例: 1644…)
THREADS_APP_SECRET Step 4 — 「表示」で取得 英数字
THREADS_ACCESS_TOKEN Step 5 — 長期トークン THAA…(150〜200文字)
THREADS_USER_ID Step 6 — User ID取得 17桁前後の数字
DISCORD_WEBHOOK_URL Step 7 — Discord Webhook https://discord.com/api/webhooks/…
✓
取得した値はその都度メニュー 🔧 初回セットアップ を再実行して、該当項目だけ入力すればOK。 他の項目は空欄のままOKを押せば、既存値はそのまま保持されます。
💡 つまずきポイント: 全部の値を一度に揃えようとすると挫折しがち。1項目取って入力 → 次への流れで進めるのがコツ。

  1. developers.facebook.com にFacebookアカウントでログイン

うまく開けない・「接続が拒否」「Navigation was blocked by Cross-Origin-Opener-Policy」が出る場合:

4-2アプリ作成

  1. 右上 マイアプリ → アプリを作成

4-3ユースケースのカスタマイズ・権限追加

アプリ作成直後は権限が空です。threads_basic と threads_content_publish を追加し、それぞれ 「テスト準備完了」 ステータスに切り替えます。

  1. マイアプリ の「管理者アプリ」から、作成したアプリをクリック
  2. ダッシュボードの 「Threads APIへのアクセス」 → カスタマイズ をクリック
  3. threads_basic と threads_content_publish を 「+ 追加」 ボタンで追加し、それぞれの ステータスを「テスト準備完了」 に切り替える
    ※ my-threads-scheduler は初期設定で「アクション」になっています

基本設定 (⚙️ 左側のアプリの設定 ➡️ ベーシック) 📋 各URLはタップでコピー

4-5APP_ID と APP_SECRET を取得 ➡️ スプシに登録 (1個ずつ実施)

  1. (左側の)ユースケース ➡️ カスタマイズ ➡️ 設定 を開く
  2. ThreadsアプリID をコピー
  3. スプレッドシートのメニュー スレッズスケジューラー ➡️ 🔧 初回セットアップ を実行
    ※ 初回の時のみ「認証が必要になります」
    1. 「このアプリはGoogleで確認されていません」 と出るが、左下の 「詳細」 をクリック
    2. 「スレッズスケジューラー(安全ではないページ)に移動」 をクリック
    3. 「アクセスできる情報を選択してください」 で 「全てを選択」 にチェックを入れて進める
    💡 「スクリプト起動しています」から動かない場合 はリロードしてからもう一度実行してください。
  4. THREADS_APP_ID を貼り付け
  5. 「Threadsのapp secret」を表示 してコピー (Facebookのパスワード入力)
  6. スプレッドシートの THREADS_APP_SECRET を入力
  7. 残りはひとまず 空欄でOKを押してスキップ
  8. 完了画面で 2項目「✅ 設定済」 表示を確認
💡 つまずきポイント: 「保存」を押しても何も起こらない場合、URL3箇所のうちどれかが空欄です。赤くハイライトされている欄を確認してください。

Threads長期トークン取得 → ACCESS_TOKEN

5-1ユーザートークン生成ツールを開く

ユースケース → Threads APIにアクセス → カスタマイズ → 設定 と進み、 ページ下部にスクロールすると 「ユーザートークン生成ツール」 があります。

5-2Threadsテスター追加または削除 + 承認

  1. 「Threadsテスターを追加または削除」 から自分のThreadsアカウントを追加
    1. 「メンバーを追加」 を選択
    2. ➡️ 「Threadsテスター」 を選択
    3. ➡️ 一番下にアカウント名を入れて選択する
    4. ➡️ 「追加」 をクリック
  2. Threadsで招待を承認する
    「Threadsユーザーは、プロフィールのウェブサイトのアクセス許可セクションから招待を管理できます」の ウェブサイトのアクセス許可 をタップし、Threadsへ移動
    1. ➡️ ウェブサイトのアクセス許可
    2. ➡️ 「招待」
    3. ➡️ 同意する
    4. ➡️ MetaのDeveloperのサイトに戻ってリロード
    5. ➡️ ステータスの 「承認待ち」が消えている ことを確認する
⚠
招待が表示されない場合: 5〜10分待つ、もしくはリロードして再読み込みしてください。

5-3長期アクセストークンを生成

  1. ユーザートークン生成ツールに戻る → ページ再読み込み
    (ユースケース ➡️ カスタマイズ ➡️ 設定 ➡️ 一番下)
  2. 承認済アカウントの右側 「アクセストークンを生成」 → 「許可」
  3. 表示されたトークン(60日有効)を コピーボタンで取得
✓
THAA で始まる長い英数字(150〜200文字)になっていればOK。
💾
ここでスプレッドシートの「🔧 初回セットアップ」に入力
  • メニュー右端の スレッズスケジューラー ➡️ 🔧 初回セットアップ を実行
  • THREADS_ACCESS_TOKEN トークンを貼り付け
  • 他の項目は 空欄でOK押下してスキップ

⚠ 長期トークンは次の Step でも使用します(再度コピーする必要はありません)。

⚠ 万が一トークンを再生成した場合は、スプレッドシートの「🔧 初回セットアップ」も新しい値に更新してください。

💡 つまずきポイント: 招待が承認できない場合、同じMetaアカウントでログインしているかを確認してください。サブアカウントでログイン中だと招待が見えません。

06
⏱ 所要時間 2分 🔢 ID取得

Threads User ID取得 → USER_ID

長期トークンを貼るとURLが生成されます。「ブラウザで開く」を押すと、トークン入りURLが Meta API (graph.threads.net) に直接送信され、JSON 表示画面が開きます。
🔒 このページや localStorage には保存されません。「ブラウザで開く」を押した時のみ Meta API に送信されます(ブラウザ履歴に残る点に注意)。

https://graph.threads.net/v1.0/me?fields=id&access_token=<長期トークン>

開くと、ブラウザに以下のようなJSONが表示されます: 

{"id":"35107700672178457"}

idの値(数字) が THREADS_USER_ID です(17桁前後)。

💾
ここでスプレッドシートの「🔧 初回セットアップ」に入力
  • メニュー右端の スレッズスケジューラー ➡️ 🔧 初回セットアップ を実行
  • THREADS_USER_ID 数字(17桁前後)を入力
💡 つまずきポイント: JSONがエラーで表示される場合、トークンの末尾に余計な空白が混入しているケースが多いです。コピーし直してください。

08
⏱ 所要時間 5分 🚀 動作確認 + 自動運用開始

テスト投稿で動作確認 → トリガーをインストール

8-1設定状況の最終確認

メニュー「スレッズスケジューラー」の 🔍 設定状況を確認 を実行 → 5項目すべてが「✅ 設定済」 になっていることを確認。

8-2テスト投稿

  1. 「投稿予約」シートに新規行を追加:
    • 📅 A列(日付): 今日の日付
    • 🕒 B列(時刻): 1〜2分前の時刻(例: 14:30)
    • 📝 C列(投稿本文): 「テスト投稿」など
    • 🏷 E列(ステータス): 「未投稿」(プルダウンで選択)
  2. M列に投稿日時が自動表示されることを確認
  3. メニュー「スレッズスケジューラー」の ▶️ 今すぐ投稿チェック を実行
  4. 数秒後に Threadsへ投稿が反映 されることを確認
  5. E列が「投稿済」 に変わることを確認
  6. Discordに「✅ 投稿成功」通知 が届くことを確認

8-3トリガーをインストール(自動運用開始)

メニュー ⏰ トリガーをインストール → 完了ダイアログ

以降、10分ごと自動チェック + 週1回トークン自動更新 が動き続けます。操作不要。

🎉
セットアップ完了です! Threadsの予約投稿が自動で動き始めます。 以降は「投稿予約」シートに行を追加するだけで運用できます。
📌 完了後の運用

セットアップ完了後の運用

  1. 新規予約投稿: 「投稿予約」シートに A=日付 / B=時刻 / C=本文 / E=未投稿 を入れるだけ
  2. 状態確認: E列ステータス + Discord通知
  3. トークン更新: 毎週日曜9時に自動実行(手動操作不要)

🛟トラブル時のセルフチェック

「投稿が飛ばない」「Discordにエラーが来た」など、何かおかしいと感じたら、下の 2つのメニュー を順番に確認してください。原因の 9割はこのどちらか で特定できます。

①
🔍 設定状況を確認

スプシ上部メニュー「スレッズスケジューラー → 🔍 設定状況を確認」をクリック。下のように5項目が表示されます。 

THREADS_APP_ID: ✅ 1234...5678
THREADS_APP_SECRET: ✅ abcd...wxyz
THREADS_ACCESS_TOKEN: ✅ THAA...XYZ9
THREADS_USER_ID: ✅ 3510...8457
DISCORD_WEBHOOK_URL: ✅ http...zzzz
  • 全部 ✅ なのに投稿が飛ばない → ②のトリガー一覧へ
  • どれか ❌ 未設定 → 🔧 初回セットアップ をもう一度実行してその項目だけ入力
  • THREADS_ACCESS_TOKEN が ❌(または60日以上更新されていない疑い)→ メニュー 🔄 トークン手動更新 を実行
②
📋 トリガー一覧

スプシ上部メニュー「スレッズスケジューラー → 📋 トリガー一覧」をクリック。正常な状態は 以下の2件 です。 

現在のトリガー: 2件

・postRunner (id=...)
・refreshTokenJob (id=...)
  • 2件揃っている → 自動投稿は正常稼働中。投稿が飛ばない場合はスプシのE列ステータス(「投稿済」「失敗」「未投稿」)を確認
  • 0件 / 1件しかない → メニュー ⏰ トリガーをインストール を再実行
  • 同じ関数名が3件以上ある → メニュー 🗑 トリガーを削除 → ⏰ トリガーをインストール でリセット
💬
それでも解決しない場合は、Discordチャンネルに通知されたエラーメッセージのスクショと一緒に、購入時のサポート窓口へお問い合わせください。
📷 オプション機能 ⏱ 設定不要・必要時のみ

画像つき投稿を使う

画像を添付したThreads投稿を予約する場合に使います。テキストだけでよい場合は不要です。 画像はあなた自身のGoogle Driveに保存され、tamago側のサーバーには一切残りません。事前設定は何もいりません。

📦仕組み

  1. スプシのメニュー 📷 画像をアップロード をクリック
  2. ダイアログで画像ファイルを選ぶ(ドラッグ&ドロップでもOK)
  3. あなたのDriveの 「スレッズ画像」フォルダ(自動作成)に保存
  4. 公開URLが「画像URL」列のセルに自動入力
  5. 投稿時刻になるとThreads APIが上記URLから画像を取得して投稿

①使い方A: メニューからアップロード

  1. 「投稿予約」シートの 「画像URL」列のセル をクリックして選択
  2. メニュー スレッズスケジューラー → 📷 画像をアップロード
  3. ダイアログに画像をドラッグ or クリックで選択 → アップロード
  4. 選択中のセルにURLが自動入力されたら完了
  5. あとは通常通り、日付・時刻・本文・ステータスを埋めるだけ

②使い方B: セル内画像をURL化

「画像URL」列のセルに画像を直接ドラッグ&ドロップで配置(Googleスプレッドシート標準の「セル内に画像を挿入」機能)した後、メニュー 🔄 セル内画像をURL化 を実行すると、配置された画像を一括でDriveに保存し、URLに置換します。複数行まとめて変換可能。

⚠️
投稿前に必ずURL化が必要: D列にセル内画像が配置されたまま投稿時刻が来ると、その行は即「エラー」になります(Threads APIは画像URL文字列しか受け付けないため)。Discord通知が届いたら「🔄 セル内画像をURL化」を実行してください。変換時にステータス・試行回数・エラーメッセージも自動でリセットされ、次のトリガー(10分以内)で自動再投稿されます。手動でステータスを戻す必要はありません。

🔍確認

メニュー 🔍 画像設定を確認 で、保存先フォルダの場所と共有設定を確認できます。

🔒
プライバシー設計: 画像はあなた自身のGoogle Drive内「スレッズ画像」フォルダに保存され、tamago側のサーバーには一切経由しません。フォルダは初回アップロード時に自動作成され、「リンクを知っている人は閲覧可」に設定されます(Threads APIが画像を取りに来るために必要)。
🗑
不要になった画像は、Drive上で「スレッズ画像」フォルダから直接削除してください。投稿済の場合、Threads側はGoogleキャッシュを参照するため、Driveから消しても投稿は残ります。
